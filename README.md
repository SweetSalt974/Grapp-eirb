# Grapp-eirb
Climbing Game made as a side project during my College 1st and 2nd year in 2023 for the Game Creation Club association **(GCC)**.

## How to Edit

You need the [4.2 Godot Editor](https://godotengine.org/download/archive/4.2-stable/), later version may still work properly with this game but is not entirely sure.

Open the Godot Editor then import the ``project.godot`` at the root of this repositories, then you can edit with all the proper godot tools.

## How to play

This game supports **AZERTY** layout and is not expected to support other layouts until reconsideration.

### Executable

Godot can export a **Windows** and **Linux** executable of Godot projects but i only tested and supported the windows version for this game.
You have the choice to export the executable yourself from Godot Editor or to download the Latest Released version.

### Game

This is a climbing Game who has up until now 2 levels. 
When launching the game press **Enter Key** and then you can choose the level with the **Arrow Directional Keys**.

Levels have an end goal (A red flag) and a set of climbing hooks than you can **Left-Click** on to dash in their direction.
Each sub-level screens can be reset upon death or by pressing the **Reset Key** (R).

You can go and find collectible in a form of cakes in the levels to fully complete them. 

You can hang on not too steep rope with **Z Key** and let go with the same Key.
While hanging you can hold **Dash Key** (space bar) to charge a dash in the mouse direction to gain further distance.

## Assets

Sound effects are gathered from royalty free libraries.
Music was made in Beepbox by myself.
In-Game pixel art assets including character design, mountain tileset, collectibles, animation and backgrounds were created using the [Pixelorama](https://pixelorama.org/) tool.
Character design was made by taking inspiration from Celeste art style.
